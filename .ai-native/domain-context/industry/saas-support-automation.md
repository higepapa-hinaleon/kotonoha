# SaaS / 企業内サポート自動化 - 業界知識

## 業界概要

組織内の問い合わせ対応（社内ヘルプデスク、カスタマーサポート）をAIで自動化する領域。RAG（Retrieval Augmented Generation）技術の実用化により、組織固有のドキュメントに基づく高精度な自動応答が可能になった。

## 業界トレンド

1. **RAGベースの社内QA:** LLMの汎用知識ではなく、組織固有のドキュメントをソースとして回答生成
2. **ハイブリッドサポート:** AI が一次対応、人間が二次対応のエスカレーションモデル
3. **ナレッジマネジメントとの融合:** サポートボットが暗黙知の形式知化を促進
4. **継続的改善ループ:** 回答できなかった質問のフィードバックによる自動学習
5. **埋め込みウィジェット:** 既存のWebサイト・ポータルに統合可能なチャットウィジェット

## 一般的な課題

| カテゴリ | 課題 | 対策 |
|---------|------|------|
| 精度 | ドキュメントにない質問への対応 | 信頼度スコアによるエスカレーション |
| 鮮度 | ドキュメントの陳腐化 | インクリメンタル更新、差分検出 |
| 導入障壁 | ドキュメント整備の初期コスト | 段階的導入、既存ドキュメントの活用 |
| 信頼性 | AIの誤回答リスク | ソース表示による透明性確保 |
| コスト | LLM API利用料 | キャッシュ、バッチ処理による最適化 |

## 専門用語

| 用語 | 説明 |
|------|------|
| RAG | Retrieval Augmented Generation - 検索拡張生成 |
| チャンキング | ドキュメントを検索可能な単位に分割する処理 |
| ベクトル埋め込み | テキストを数値ベクトルに変換し、意味的類似度検索を可能にする |
| 信頼度スコア | AI回答の確からしさを示す数値指標 |
| エスカレーション | AIで対応できない問い合わせを人間に引き継ぐこと |
| ナレッジベース | 組織の知識を体系的に蓄積・管理するデータベース |
| コンテキスト付きリトリーバル | チャンクにドキュメント内の文脈情報を付与して検索精度を向上させる手法 |
