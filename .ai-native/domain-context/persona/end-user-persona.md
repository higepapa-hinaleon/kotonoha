# エンドユーザー ペルソナ

## 基本情報

| 項目 | 内容 |
|------|------|
| 対象 | 組織内の一般社員・メンバー |
| 役職 | 問い合わせを行う全社員 |
| ITリテラシー | 中程度（Webブラウザ・チャットツールの操作に慣れている） |
| 利用頻度 | 週1-5回（業務上の疑問発生時） |

## 利用シーン

- **いつ:** 業務時間中に不明点が生じた時。特に営業時間外の夜間・休日。
- **どこで:** デスク（PC）またはモバイルデバイス。外部サイトの埋め込みウィジェット経由の場合もある。
- **何のために:** 業務手順・社内ルール・製品仕様などの確認。従来は担当者へのメール・チャットで問い合わせていた内容。

## ペインポイント

1. **回答待ち時間:** 担当者に問い合わせると数時間〜翌日まで待つことがある
2. **誰に聞けばいいかわからない:** 知識が属人化しており、適切な問い合わせ先がわからない
3. **回答品質のばらつき:** 担当者によって回答の正確さ・詳しさが異なる
4. **同じ質問の繰り返し:** 以前聞いた内容を忘れて再度問い合わせる
5. **情報の散在:** マニュアル・FAQ・社内Wiki等、情報源が分散している

## システムに対する期待値

- 即座に回答が得られること（秒単位）
- 回答の根拠（参照元ドキュメント）が確認できること
- 24時間365日いつでも利用可能であること
- 回答できない場合は適切な担当者・窓口に誘導されること
- 過去の問い合わせ履歴を振り返れること
