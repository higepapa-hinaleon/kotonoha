# 管理者 ペルソナ

## 基本情報

| 項目         | 内容                                                   |
| ------------ | ------------------------------------------------------ |
| 対象         | サポートチーム管理者、ナレッジマネージャー             |
| 役職         | チームリーダー、マネージャー、情報システム部門担当者   |
| ITリテラシー | 中〜高（管理画面の操作、ドキュメント管理に慣れている） |
| 利用頻度     | 毎日（モニタリング）、週次（レポート確認・改善対応）   |

## 利用シーン

- **毎日:** ダッシュボードで問い合わせ状況を確認。未解決の改善リクエストを処理。
- **週次:** 週次レポートを確認し、トレンドと改善ポイントを分析。FAQを更新。
- **随時:** 新規ドキュメント登録、サービス設定変更、ボット設定調整。

## ペインポイント

1. **問い合わせ対応コストの増大:** 同じ質問への繰り返し回答に工数がかかる
2. **ナレッジの属人化:** 退職・異動で知識が失われるリスク
3. **改善サイクルの欠如:** どの質問が多いか、何が回答できていないかが可視化されていない
4. **ドキュメントの散在・陳腐化:** 情報が複数の場所に分散し、最新性が保てない
5. **対応品質の管理困難:** 個別の対応品質を把握・統制する手段がない

## システムに対する期待値

- ドキュメントを登録するだけでAIが回答可能になること
- 回答できなかった質問が自動的に改善リクエストとして上がること
- ダッシュボードで全体状況を一目で把握できること
- FAQの自動生成で管理負担が軽減されること
- 週次レポートで改善の方向性が示されること
- 設定変更（信頼度閾値等）が容易に行えること
